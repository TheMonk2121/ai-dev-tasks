## 🔧 Detailed Setup & Installation

### **Prerequisites**
- **Python 3.12+**: Latest Python version for optimal performance and latest language features
- **PostgreSQL 14+**: Database for vector storage and metadata with PGVector extension
- **Node.js 18+**: For N8N workflow automation and modern JavaScript features
- **Git**: Version control and collaboration
- **Docker** (Optional): For containerized deployment and development
- **Make** (Optional): For simplified build and deployment commands

### **Environment Setup**

#### **1. Database Configuration**
```bash
# Install PostgreSQL and PGVector
# Ubuntu/Debian
sudo apt-get update
sudo apt-get install postgresql postgresql-contrib
sudo apt-get install postgresql-14-pgvector

# macOS
brew install postgresql
brew install pgvector

# Windows (using WSL or Docker)
# Use Docker: docker run --name postgres-pgvector -e POSTGRES_PASSWORD=password -p 5432:5432 pgvector/pgvector:pg14

# Create database and user
sudo -u postgres createdb ai_dev_tasks
sudo -u postgres createuser ai_user
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE ai_dev_tasks TO ai_user;"
sudo -u postgres psql -c "ALTER USER ai_user WITH PASSWORD 'your_secure_password';"

# Enable PGVector extension
sudo -u postgres psql -d ai_dev_tasks -c "CREATE EXTENSION IF NOT EXISTS vector;"
```

#### **2. Python Environment**
```bash
# Create virtual environment
python3.12 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Upgrade pip and install dependencies
pip install --upgrade pip
pip install -r requirements.txt
pip install -r requirements-constraints.txt

# Install development dependencies (optional)
pip install -r requirements-dev.txt

# Verify installation
python3.12 -c "import dspy; print('DSPy installed successfully')"
python3.12 -c "import psycopg2; print('PostgreSQL driver installed successfully')"
```

#### **3. Configuration**
```bash
# Copy configuration template
cp config/config.example.py config/config.py

# Edit configuration
nano config/config.py  # or use your preferred editor

# Set environment variables
export POSTGRES_DSN="postgresql://ai_user:your_secure_password@localhost:5432/ai_dev_tasks"
export N8N_BASE_URL="http://localhost:5678"
export N8N_API_KEY="your_api_key_here"

# For Windows (PowerShell)
# $env:POSTGRES_DSN="postgresql://ai_user:your_secure_password@localhost:5432/ai_dev_tasks"
# $env:N8N_BASE_URL="http://localhost:5678"
# $env:N8N_API_KEY="your_api_key_here"

# Create .env file for persistent environment variables
cat > .env << EOF
POSTGRES_DSN=postgresql://ai_user:your_secure_password@localhost:5432/ai_dev_tasks
N8N_BASE_URL=http://localhost:5678
N8N_API_KEY=your_api_key_here
EOF
```

#### **4. N8N Workflow Setup**
```bash
# Install N8N globally
npm install -g n8n

# Start N8N (development mode)
n8n start

# For production, use PM2 or Docker
# PM2: pm2 start n8n --name "ai-dev-tasks-n8n"
# Docker: docker run -it --rm --name n8n -p 5678:5678 n8nio/n8n

# Import workflows
# Navigate to http://localhost:5678 and import workflows from config/n8n/
# Key workflows to import:
# - Backlog Scrubber Workflow
# - AI-BACKLOG-META Processor
# - Document Processing Pipeline
```

### **🔧 Customizing AI Agents**
Configure specialized agents for your needs:

```python
# Configure agent roles
AGENT_CONFIG = {
    "planner": {
        "model": "cursor-native-ai",
        "context_files": ["000_core/000_backlog.md", "400_guides/400_system-overview.md"],
        "temperature": 0.7,
        "max_tokens": 2000
    },
    "implementer": {
        "model": "cursor-native-ai",
        "context_files": ["400_guides/400_comprehensive-coding-best-practices.md"],
        "temperature": 0.3,
        "max_tokens": 4000
    },
    "researcher": {
        "model": "cursor-native-ai",
        "context_files": ["500_research/500_research-index.md"],
        "temperature": 0.8,
        "max_tokens": 3000
    }
}

# Use agents in your workflow
from scripts.agent_manager import AgentManager
agent_manager = AgentManager(AGENT_CONFIG)

# Get planner response
planner_response = agent_manager.get_planner().plan_feature("user authentication")

# Get implementer response
implementer_response = agent_manager.get_implementer().implement_feature(planner_response)

# Multi-role consensus
consensus = agent_manager.get_consensus(["planner", "implementer", "researcher"])
```
